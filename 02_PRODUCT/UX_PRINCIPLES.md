# UX Principles

Last updated: 2026-04-06
Status: active

## Purpose

Define the interaction philosophy for DNA Film Content Engine so the interface remains aligned with the product's meaning-first logic.

## P-001 Meaning first

The interface should treat semantic structure as the primary control layer.

### Consequence
The core user surface is semantic-map control, not raw timeline manipulation.

## P-002 One project, multiple outputs

The product should feel like one project that branches into outputs, not like separate disconnected tools.

### Consequence
Long video, shorts, carousel, and packaging should all read as views over the same project source.

## P-003 Strong system proposal, human approval

The system may propose structure and later automation, but the user remains the final approver.

### Consequence
Every major transformation should be reviewable.

## P-004 Desktop-first scanning

The layout should prioritize scan speed, comparison, and block-level control.

### Consequence
The interface should prefer panes, inspectors, summaries, and visible stage status over modal-heavy flows.

## P-005 Visible readiness

Every screen should make the project's current readiness and blockers legible.

### Consequence
The user should always know what is ready, what is missing, and what the next meaningful action is.

## P-006 Bounded future visibility

Future lanes may be visible for orientation, but must not pretend to be active when they are not implemented.

### Consequence
Use explicit placeholder states rather than fake completeness.

## P-007 Reduce automation theater

The product should not overclaim autonomy where quality still requires editorial judgment.

### Consequence
The interface should make approval and revision natural instead of hiding control behind pseudo-magic.

## P-008 Product language should match the real job

The UI should speak in terms of analysis, blocks, scenes, outputs, and packages, not in generic editor jargon where it reduces clarity.

### Consequence
Navigation and labels should reflect the DNA workflow rather than imitate generic video tools.

## P-009 Prepared growth without early overload

The MVP should prepare for later matching and output stages without dragging them fully into the first working surface.

### Consequence
Early screens should be stable and deep enough to matter, while later screens remain bounded.
