# MVP vs Full

Last updated: 2026-04-06
Status: active

## MVP

The MVP should prove one coherent desktop-first product loop:

1. create project;
2. load analysis text;
3. break text into semantic blocks;
4. review and refine the semantic map in a dedicated interface;
5. prepare output-oriented structure for later scene matching.

### MVP focus
- usability of the editor-facing workflow;
- clarity of navigation;
- semantic control and visibility;
- strong product structure for later vertical slices.

## Full version

The full version can extend into:
- film ingest and transcript alignment;
- scene matching;
- timecode candidate generation;
- rough-cut generation;
- shorts/reels generation;
- carousel generation;
- packaging and export center;
- reusable film library;
- style learning and editor preference memory.

## Rule

No later-phase automation should be pulled into MVP unless it directly serves the first coherent desktop user workflow.
